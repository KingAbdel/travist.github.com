Automating the Web With jQuery + Node.js + Phantom.js

Aug 20th, 2013

As most of you may already know (from my previous articles), I have taken a deep interest in creating web automation and testing tools using a combination of Node.js and Phantom.js browser, which combined give you the power to build some really amazing automation tools. Previously, my focus has been to leverage existing interfaces that has already taken root in web automation within the Node.js community. Specifically, these interests were in providing the Zombie.js interface to the Phantom.js browser within Node.js. This effort was captured in a previous blog post that you are welcome to read, but I am not here to talk about that… to be honest, I was not particularly happy with how it turned out simply because I realized that the Zombie.js interface is not the most elegant of interfaces for reasons out of the scope of this article.

So when it comes to Javascript interfaces, there is really one that stands out as being the defacto standard in web development… jQuery.

jQuery’s separation between “Find Something” and “Do Something” combined with method chaining was such a simple but powerful notion that literally changed the way we all write JavaScript. The following command seems so simple these days…

However, when this first came out, it was revolutionary. People just ‘got it’ and it made working with JavaScript fun again. So, why do we not have the same jQuery interface to build web automation and testing tools? The interfaces that we get with web automation tools are those of Selenium and others, but these must be learned… They don’t seem familiar… at least not to me.

So, I sought out to look for a web automation tool that used the jQuery interface within Node.js. I came up empty handed. I did find the ‘Official’ jQuery project for Node.js, but this was not used for web automation. In fact, it is really hard to figure out what it IS used for… I think you could use it to help parse through DOM’s within Node.js or even XML, but web automation and testing… not so much. So, I did what any other software geek would do in this situation… I created my own. I am calling it jquery.go.js and I am pretty happy how it turned out…
